# Archive — historical build logs & superseded specs

These documents describe **how the app was originally planned and built**, phase by phase. They
are kept for narrative history and are **not** maintained — treat them as point-in-time records
that may be stale. The durable rationale has been folded into the live docs; for the actual
change history, use `git log`.

**For current truth, use the live docs instead:**
- What the app is & how it's wired → [`../ARCHITECTURE.md`](../ARCHITECTURE.md) (+ [`../BUDGET.md`](../BUDGET.md), [`../SALARY.md`](../SALARY.md), [`../DESKTOP.md`](../DESKTOP.md))
- Possible future work → [`../IDEAS.md`](../IDEAS.md)

**Contents:**
- `SPEC.md` — the original concept/intent spec (the "what & why"). Superseded by the live docs.
- `PLAN.md` — the original design & phased build plan (the "how"). Superseded by `ARCHITECTURE.md`.
- `SALARY_SPEC.md` / `SALARY_PLAN.md` — the original salary-tab design + 8-task build log. Superseded by `SALARY.md`.
- `DESKTOP_SPEC.md` / `DESKTOP_PLAN.md` — the desktop (Tauri) design + build log. Superseded by `DESKTOP.md`.
- `IMPROVEMENTS.md` — a 2026-06-14 review punch-list (many items since built or overtaken); survivors moved to `IDEAS.md`.
- `superpowers/` — design specs + implementation plans from brainstorming/planning sessions: the
  2026-06 salary-redesign work (sub-tabs, lifetime, student-loan; superseded by `SALARY.md`) and
  the 2026-07 Overview category filter + Views work (superseded by `BUDGET.md`).
